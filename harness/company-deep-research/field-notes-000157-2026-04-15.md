# Field Notes (000157.SZ, 2026-04-15)

This note records what broke or felt awkward when running `company-deep-research` in this environment, and what should be added to the skill for reuse.

## What Worked

- CNINFO hosts the full annual report PDF at a stable `static.cninfo.com.cn` URL; it is a strong primary source for key financial tables, segment revenue, overseas share, and risk disclosures.
- The cycle framework can be grounded using association monthly sales numbers (often reprinted by major portals).

## Issues Encountered

- Missing local artifacts:
  - `/Users/rappy/.openclaw/workspace-stockyy/stock/` exists but was empty here, so no prior report/CSV could be reused.
- Latest close retrieval failed:
  - Running `/Users/rappy/.openclaw/workspace-stockyy/scripts/get_latest_close.py` failed with `No module named 'akshare'`.
  - The default Python here is `/opt/anaconda3/bin/python3` (Python 3.13), which may not match `akshare` support expectations.
- Deep-research execution ambiguity:
  - The instruction “use Claude-deep-research” was interpreted as multi-source web cross-checking, but there is no deterministic on-box “deep-research tool” in this Codex CLI runtime.
- K-line / MA analysis limitation:
  - Without a local data SDK (AkShare) or a stable quote export path, it was hard to compute MA/ATR/RSI from completed bars.
  - Result: technical section had to be written conservatively with explicit data gaps.

## Reusable Fixes to Add (v2 candidates)

- Preflight checks section:
  - Check Python interpreter and dependency availability before promising “latest close” retrieval.
  - If `akshare` missing, fall back to a documented web-based quote source and mark it clearly.
- Primary-source-first checklist:
  - Always fetch CNINFO annual report PDF and extract: revenue/profit/OCF, segment revenue, domestic vs overseas, key risk disclosures, leverage ratios.
- Data caching convention:
  - Save the annual report PDF and a `key_figures.json` snapshot under a deterministic path like `data/<ticker>/<report-date>/`.
- K-line fallback workflow:
  - If API data fetch fails, require user-provided CSV export (or documented scrape path) and run a local script to compute MAs.
- “Latest close” integrity rule:
  - If the true latest trading-day close cannot be verified, the output must explicitly say “latest verified close = …; latest trading day close pending verification”.
